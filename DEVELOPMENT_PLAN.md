## 1. 项目概述

CrabCoder 是一款基于 Go 和 Bubble Tea 框架的 AI 编程助手 CLI 工具，支持多模型、流式输出、插件扩展。

### 核心目标

- **快速启动**: 冷启动 < 500ms
- **低延迟**: AI 响应延迟 < 100ms
- **安全可靠**: 权限控制、命令审计
- **可扩展**: OpenSpec + 插件系统

## 12. OpenSpec 规范驱动开发

根据架构文档中的 OpenSpec 设计，建议采用 TDD 开发流程：

```
┌─────────────────────────────────────────────────────────────┐
│                    OpenSpec + TDD 流程                       │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  1. 编写 Spec (tool.yaml)                                   │
│         ↓                                                   │
│  2. 生成测试模板 (openspec generate)                        │
│         ↓                                                   │
│  3. Red: 编写失败的测试                                      │
│         ↓                                                   │
│  4. Green: 实现最小代码通过测试                               │
│         ↓                                                   │
│  5. Refactor: 优化代码质量                                   │
│         ↓                                                   │
│  6. 验证 Spec 与实现一致                                     │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 工具开发顺序 (建议)

基于依赖关系，建议按以下顺序开发工具：

1. `read` - 最基础的读取工具
2. `write` - 创建/覆盖文件
3. `edit` - diff 格式编辑
4. `glob` - 文件搜索
5. `grep` - 内容搜索
6. `bash` - 命令执行
7. `delete` - 删除文件/目录
